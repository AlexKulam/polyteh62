Программа моделирует очередь студентов к компьютерам с использованием класса SemaphoreSlim.

В программе есть 10 студентов и 3 компьютера. Одновременно работать за компьютерами могут только три студента, остальные ожидают своей очереди до освобождения ресурса.

Каждый студент запускается как отдельная задача. После получения доступа к компьютеру студент выполняет работу некоторое время, а затем освобождает компьютер для следующего ожидающего студента.

Для ограничения количества одновременно работающих студентов используется SemaphoreSlim. Он позволяет контролировать доступ к общему ресурсу и предотвращает превышение установленного количества подключений.

Цель работы — изучить механизм синхронизации потоков и ограничение количества одновременно выполняющихся задач с помощью SemaphoreSlim в C#.
